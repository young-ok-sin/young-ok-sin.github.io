# 한국전력기술 경험 상세 페이지 설계

## 목표

포트폴리오 메인 페이지의 `Experience` 섹션에 있는 한국전력기술 경험 카드를 상세 페이지로 연결하고, 상세 페이지에서 참여 프로젝트 설명과 PDF 자료를 새 탭으로 바로 열 수 있게 구성한다.

## 현재 맥락

- 메인 페이지는 [index.html](/c:/Users/SSAFY/Desktop/portfolio/young-ok-sin.github.io/index.html)의 `Experience` 섹션에서 경력 카드를 노출한다.
- 프로젝트 상세 페이지는 별도 HTML 파일로 분리되어 있으며, 기존에 `moa.html`, `healthcare-ai.html`, `sebook.html` 패턴이 있다.
- 상세 페이지는 기존 스타일 패턴을 재사용하는 구조가 적합하다.

## 승인된 방향

- 한국전력기술 항목은 `Selected Projects`가 아니라 `Experience` 섹션의 카드에서 상세 페이지로 진입한다.
- PDF는 다운로드가 아니라 브라우저에서 새 탭으로 바로 연다.
- 상세 페이지는 기존 케이스 스터디 페이지처럼 별도 HTML 파일 하나로 구성한다.
- PDF 파일은 저장소 내부 경로로 복사해 GitHub Pages 배포 환경에서도 접근 가능해야 한다.

## 정보 구조

### 메인 페이지

- `Experience` 섹션의 한국전력기술 카드를 클릭 가능한 링크 카드로 변경한다.
- 카드에는 기간, 역할, 핵심 업무 요약만 유지한다.
- 상세 설명은 메인 카드에 과도하게 넣지 않고 상세 페이지에서 읽도록 분리한다.

   - 핵심 키워드

   - 문서 OCR 관련 프로젝트

3. 문서 OCR 관련 프로젝트
   - 진행 내용
   - 진행 과정
   - 라벨링 결과 분석 PDF 링크
4. 도메인 특화 검색 시스템 개발
   - 진행 내용

## 콘텐츠 반영 범위

### 1. 문서 OCR 관련 프로젝트

- 문서 라벨링 진행
- 라벨링 결과를 토대로 hyperParameter 튜닝
- 여러 OCR 라이브러리를 사용하며 결과 비교
- 성능 향상을 위해 table line 제거 전처리 과정 개선
- OCR 라이브러리 hyperParameter 튜닝을 통한 성능 향상

진행 과정에는 PDF 자료 링크를 포함한다.

### 2. 도메인 특화 검색 시스템 개발

- 외부 데이터 파일(pdf)들을 문장 단위로 저장하는 코드 개발
- GitHub 저장소 링크: `https://github.com/young-ok-sin/pdf-extractor`

### 이외 잔업

- pandas를 활용해 excel 파일의 데이터를 가공하여 저장하는 업무 수행

## 파일 및 자산 배치

- 새 상세 페이지 파일을 추가한다. 파일명은 `kepco-enc.html`로 둔다.
- PDF 파일은 저장소 내부의 정적 경로에 복사한다.
- PDF 파일명은 웹 경로 안정성을 위해 영문 기반 이름을 사용한다.
- 메인 페이지와 상세 페이지 모두 상대경로로 연결해 로컬 미리보기와 GitHub Pages 배포에서 동일하게 동작하게 한다.

## 링크 동작

- 한국전력기술 경험 카드 클릭 시 상세 페이지로 이동한다.
- PDF 링크는 `target="_blank"`와 `rel="noreferrer"`를 사용해 새 탭에서 바로 연다.
- GitHub 링크도 새 탭으로 연다.

## 스타일 및 UI 원칙

- 기존 상세 페이지와 동일한 정보 밀도와 읽기 흐름을 유지한다.
- 현재 포트폴리오의 카드형 시각 언어를 유지한다.
- 모바일에서도 제목, 리스트, 링크 영역이 무너지지 않도록 기존 반응형 스타일을 따른다.
- 필요한 경우 기존 상세 페이지 스타일 파일을 재사용하거나 최소 범위로 확장한다.

## 검증 기준

- 메인 페이지에서 한국전력기술 카드 클릭 시 상세 페이지로 이동한다.
- 상세 페이지에서 PDF 링크 클릭 시 새 탭으로 PDF가 열린다.
- 상세 페이지에서 GitHub 링크 클릭 시 저장소가 열린다.
- 모바일 폭에서도 섹션 순서와 카드 레이아웃이 읽기 가능해야 한다.
- 배포 환경에서 로컬 절대경로가 아닌 저장소 상대경로만 사용한다.
