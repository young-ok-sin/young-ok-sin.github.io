Overview

이 페이지는 기존 포트폴리오의 프로젝트 상세 페이지를 보다 문서형으로 보여주기 위한 샘플입니다. 핵심 목표는 “프로젝트를 설명하는 카드”가 아니라 “GitHub README를 읽는 경험”에 가깝게 만드는 것입니다.

프로젝트 소개, 설치 방법, 디렉터리 구조, 트러블슈팅, 회고까지 한 번에 읽히는 흐름을 만드는 데 집중했습니다.

본문은 하나의 긴 문서로 이어지고, 중요한 정보는 배지와 목차, 표, 코드 블록 같은 README 익숙한 장치로 정리합니다.

Tech Stack

Area Stack Reason
Backend Java, Spring Boot API, 비즈니스 로직, 상태 전이 구조를 설명하기 좋음
Data MySQL, Redis 저장과 조회 흐름을 README 안에서 구분하기 쉬움
Infra AWS S3, GitHub Actions 배포 및 정적 자산, 문서 자동화 예시를 함께 보여주기 좋음

Features

  • 프로젝트 목적과 역할을 문서 상단에서 바로 이해할 수 있도록 구성
  • 설치 방법과 디렉터리 구조를 포함해 실제 README처럼 읽히는 본문 흐름 제공
  • 코드 블록, 표, 인용문, 체크리스트를 함께 사용해 텍스트만 늘어지지 않게 구성
  • 포트폴리오 컨텍스트는 유지하되, 카드보다 문서 경험을 우선시함

Troubleshooting

문제

이전 샘플은 README 내용을 카드로 잘게 나눈 형태라, GitHub README를 읽는 익숙한 경험과는 거리가 있었습니다.

해결

본문을 하나의 문서로 통합하고, 제목 계층과 표, 인용문, 설치 코드 블록, 디렉터리 구조 같은 README 핵심 요소를 전면으로 올렸습니다.

결과

지금은 프로젝트 소개 페이지라기보다 실제 저장소의 README를 읽는 느낌에 훨씬 가깝습니다.

Retrospective

  • 프로젝트를 처음 보는 사람도 문서 흐름만 따라가면 구조를 이해할 수 있어야 한다.
  • 카드 디자인보다 README 문법의 위계를 먼저 보여준다.
  • 배지, 목차, 표, 코드 블록, 체크리스트는 과하지 않게 핵심만 남긴다.
  • 실제 프로젝트에 적용할 때는 링크와 수치를 실데이터로 교체하면 된다.
