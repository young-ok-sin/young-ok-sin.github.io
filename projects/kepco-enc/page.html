Project

KEPCO

한국전력기술 현장실습에서 수행한 OCR 품질 분석과 Paper PDF Extractor 전처리 파이프라인 작업입니다. 핵심은 PDF 원문을 검색형 문장 데이터로 바꾸는 흐름을 설계하는 것이었습니다.

Overview

현장실습에서는 문서 처리와 검색형 데이터 구조화 작업을 중심으로 실무를 경험했습니다. OCR 라벨링 결과를 분석하고, PDF 문서를 RAG에 적합한 문장 데이터로 바꾸는 Paper PDF Extractor 흐름을 정리했습니다.

단순 추출이 아니라 줄바꿈 복구, 노이즈 제거, 문장 분리, 저품질 문서 필터링까지 포함한 전처리 설계가 핵심이었습니다.

  • Paper PDF Extractor 흐름으로 PDF를 검색형 문장 데이터로 변환

개발 내용

OCR 라벨링 결과를 분석해 품질 기준 정리

문서 인식 성능을 막연히 비교하지 않고, 실제 라벨링 결과를 바탕으로 어떤 구간에서 오류가 발생하는지 분류하는 작업부터 진행했습니다.

  • 문서 유형별 OCR 결과를 검토하고 오류가 자주 발생하는 패턴을 정리했습니다.
  • 라벨링 결과를 바탕으로 품질 개선이 필요한 영역을 구체적으로 도출했습니다.
  • 단순 정확도 비교보다 후속 검색 처리에 적합한 결과인지까지 함께 판단했습니다.

Paper PDF Extractor 전처리 파이프라인 구성

PDF 원문을 그대로 쓰지 않고, 검색과 임베딩에 적합한 문장 데이터로 바꾸는 전처리 흐름을 파이프라인 형태로 정리했습니다.

  • 텍스트 추출, 줄바꿈 복구, 노이즈 제거, 문장 분리 단계를 순서대로 연결했습니다.
  • 문장 단위 저장이 가능하도록 CSV 구조를 설계해 후속 활용도를 높였습니다.
  • 추출 도구가 아니라 데이터셋 생성 파이프라인으로 설명될 수 있도록 처리 단계를 명확히 했습니다.

저품질 문서 필터링과 결과물 정리

전처리 결과를 그대로 넘기지 않고, 실제 검색형 데이터로 사용할 수 있는 수준까지 품질을 걸러내는 단계를 추가했습니다.

  • 노이즈가 많은 문서를 후속 단계에서 제외할 수 있도록 필터링 기준을 보강했습니다.
  • 분석 문서와 코드 결과를 함께 정리해 실무 보고 형태로 남길 수 있게 했습니다.
  • RAG 전처리 단계에서 어떤 품질 보정이 필요한지 실무 관점으로 정리했습니다.

Retrospective

  • OCR 성능은 모델 선택보다 입력 문서 유형 이해가 먼저다.
  • RAG 품질은 전처리 단계에서 크게 좌우된다.
  • 실무 문서는 코드와 결과물을 함께 묶어야 설명력이 높아진다.
