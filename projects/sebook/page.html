Project

SEBook

독서 데이터를 기반으로 recommendation 기능을 제공하면서, 사용자 간 community 상호작용도 함께 설계한 프로젝트입니다. 팀장과 백엔드 총괄 역할을 맡아 서비스의 핵심 구조를 정리했습니다.

Overview

SEBook은 독서 기록과 취향 데이터를 바탕으로 도서를 추천하고, 사용자 간 감상을 공유할 수 있도록 만든 서비스입니다. 추천 결과가 고립되지 않도록 community 기능을 함께 설계한 점이 특징입니다.

이 프로젝트의 핵심은 단순 추천보다 “추천 이후의 상호작용”까지 연결하는 것이었습니다.

Tech Stack

Area Stack Reason
Backend Django, MySQL 게시판과 추천 데이터 흐름을 빠르게 구현
Recommendation cosine similarity, vector features 도서 제목, 작가, 설명 기반 recommendation 계산
Community post, review, interaction 사용자 활동이 추천 경험과 이어지도록 설계

Key Features

  • recommendation 로직으로 유사한 도서를 탐색
  • community 게시판에서 독후감과 인상 깊은 문장을 공유
  • 저장 도서와 유사도 계산을 연결해 추천 흐름 구성
  • 팀장 역할로 데이터 수집, DB 구조, 기능 우선순위 조율
book metadata
-> vectorize title / author / summary
-> calculate similarity
-> blend save-count signal
-> deliver recommendation

개발 내용

도서 메타데이터 기반 추천 로직 설계

추천 결과가 단순 조회가 아니라 서비스 핵심 경험이 되도록 제목, 저자, 설명 데이터를 특징값으로 정리해 추천 흐름을 만들었습니다.

  • 도서 제목, 저자, 설명을 기반으로 유사도를 계산하는 추천 구조를 설계했습니다.
  • 저장 수와 같은 서비스 내부 신호를 함께 반영해 추천 결과가 실제 사용 흐름과 연결되도록 했습니다.
  • 추천 로직이 독립된 계산으로 끝나지 않도록 서비스 화면과 연결되는 반환 구조를 정리했습니다.

추천과 커뮤니티가 이어지는 사용자 흐름 구성

추천 결과를 본 뒤 사용자가 바로 감상과 후기를 남길 수 있도록 커뮤니티 기능을 함께 설계했습니다.

  • 게시글과 리뷰 기능이 추천 경험과 분리되지 않도록 커뮤니티 구조를 함께 설계했습니다.
  • 추천 결과에서 사용자 상호작용으로 이어지는 흐름을 고려해 화면과 API 연결을 정리했습니다.
  • 서비스가 단순 추천기가 아니라 독서 경험을 공유하는 구조로 보이도록 기능을 묶었습니다.

초기 데이터 수집과 DB 구조를 함께 재정리

OpenAPI 활용 경험이 적은 상태에서 시작했기 때문에, 데이터 수집 방식과 DB 구조를 먼저 정리해 이후 기능 구현이 흔들리지 않게 했습니다.

  • 외부 데이터 수집 계획을 먼저 세우고 필요한 필드 중심으로 테이블 구조를 다시 정리했습니다.
  • 추천 기능과 community 기능이 함께 작동할 수 있도록 데이터 관계를 맞췄습니다.
  • 프로젝트 초기에 기능보다 데이터 흐름을 먼저 정리해 이후 구현 속도를 높였습니다.

Retrospective

  • 추천 시스템은 데이터 수집 계획이 설계만큼 중요하다.
  • community 기능은 추천 경험을 지속시키는 장치로 유효했다.
  • 초기 DB 설계에서 후속 기능을 충분히 상정해야 한다.
