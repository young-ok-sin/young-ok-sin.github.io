Project

Healthcare AI

블록체인 기반 의료 데이터 인증과 외부 의료 API 연동을 함께 다룬 프로젝트입니다. 팀장과 백엔드 개발 역할을 동시에 맡아 서비스 방향과 구현을 함께 이끌었습니다.

Overview

프로젝트 목표는 의료 데이터의 신뢰성을 높이면서도, 노약자 맞춤형 편의 기능과 질의응답 경험을 함께 제공하는 것이었습니다. 저는 회원 관리, 외부 의료 API 통신, blockchain 전송 로직, 원격 서버 운영까지 맡았습니다.

기술 포인트는 단일 기능 구현보다 “의료 데이터가 어떻게 들어오고, 검증되고, 저장되는가”를 일관되게 연결하는 것이었습니다.

Tech Stack

Area Stack Reason
Backend spring-boot, MySQL 회원 관리, 의료 API 연동, 데이터 저장 구조 구현
Integration external api, blockchain 의료 데이터 수집과 해시 기반 전송 흐름 구성
Ops GitHub, remote server 팀 운영과 실제 실행 환경 관리

Key Features

  • 사용자 계정 관리와 인증 관련 백엔드 구현
  • 외부 의료 API 통신과 데이터 수집 흐름 연결
  • 수집한 정보를 hash 처리해 blockchain 기록 흐름으로 전송
  • 원격 서버 운영과 팀 개발 일정 조율 병행
request medical data
-> validate member
-> transform to hash
-> send to blockchain
-> persist service state

개발 내용

회원과 인증 흐름을 서비스 기준으로 정리

기본 사용자 기능이 흔들리지 않도록 인증과 회원 관리 로직을 먼저 정리하고, 다른 의료 기능이 그 위에서 동작하도록 구성했습니다.

  • 회원 도메인과 인증 처리 흐름을 백엔드 기준으로 분리해 기본 진입 경로를 안정화했습니다.
  • 외부 의료 데이터 요청 전에 필요한 검증 단계를 서비스 로직 안에서 선행하도록 구성했습니다.
  • 프로젝트 전체 기능이 사용자 기준 시나리오로 이어지도록 기본 상태와 책임 범위를 먼저 맞췄습니다.

의료 데이터 수집과 변환 파이프라인 연결

외부 의료 API에서 받은 데이터를 바로 저장하지 않고, 서비스에서 사용할 수 있는 형태로 변환한 뒤 다음 단계로 넘기는 구조를 만들었습니다.

  • 외부 API 통신, 응답 해석, 내부 데이터 구조 변환을 단계별로 분리했습니다.
  • 의료 데이터 수집 흐름이 다른 도메인 로직과 섞이지 않도록 처리 책임을 나눴습니다.
  • 나중에 blockchain 전송 단계로 이어질 수 있도록 데이터 흐름을 일관된 순서로 정리했습니다.

blockchain 기록과 팀 운영 흐름을 함께 정리

데이터 전송 자체만 구현하는 데 그치지 않고, 팀장 역할로 일정과 우선순위를 조정해 구현 범위가 흐트러지지 않도록 관리했습니다.

  • 수집한 정보를 hash 기반으로 변환해 blockchain 기록 흐름에 연결했습니다.
  • 백엔드 구현 범위와 프로젝트 목표를 맞추기 위해 작업 단계를 서비스 흐름 기준으로 정리했습니다.
  • 원격 서버 운영과 일정 조율까지 병행해 팀 결과물이 같은 방향으로 수렴하도록 이끌었습니다.

Retrospective

  • 외부 시스템이 늘어날수록 책임 분리가 먼저 필요하다.
  • 팀장 역할과 구현 역할을 함께 할 때는 우선순위 정리가 특히 중요하다.
  • 기능 설명보다 데이터 흐름 설명이 이 프로젝트에서는 더 중요했다.
