Paper PDF Extractor 전처리 파이프라인

영어 PDF 원문을 RAG에 적합한 검색형 문장 데이터로 변환하는 전처리 파이프라인

paper_extractor.py는 PDF 문서를 RAG(Retrieval-Augmented Generation)의 기초 데이터로 사용할 수 있도록 정제하고 구조화하는 전처리 스크립트입니다. 단순 텍스트 추출에 그치지 않고, 줄바꿈으로 깨진 단어 복구, 불필요한 기호 제거, 문장 단위 분리, 저품질 문서 필터링까지 수행해 검색과 임베딩에 더 적합한 형태의 문장 데이터를 생성합니다.

현재 파일 구조

  • paper_extractor.py: 현재 실제로 사용하는 메인 PDF 추출 스크립트
  • temp/book_extractor_unused.py: 사용하지 않는 임시 보관용 파일

왜 필요한가

RAG 시스템의 성능은 검색 모델이나 LLM 이전에 입력 문서 품질에 크게 영향을 받습니다. 특히 PDF 원문은 줄바꿈으로 분리된 단어, 표와 수식의 혼재, OCR 결과의 노이즈, 깨진 문장 경계 같은 문제를 자주 포함합니다. 이 프로젝트는 문서를 정제하고 문장 단위로 저장해 후속 검색 단계에 바로 쓸 수 있도록 만드는 데 초점을 맞췄습니다.

RAG 전처리 흐름

  • PDF 수집 및 배치 처리: 폴더 내 PDF를 순차적으로 처리하고 결과 CSV와 제외 문서 CSV를 생성
  • PDF 로딩과 예외 처리: 손상 문서는 개별적으로 기록하고 전체 배치는 계속 진행
  • 텍스트 추출: 페이지 텍스트를 안전하게 가져온 뒤 전처리를 바로 적용
  • 노이즈 제거와 텍스트 복구: 줄바꿈 복구, 하이픈 처리, 반복 기호와 빈 괄호 제거
  • 검색 단위 생성: 문장 단위로 저장해 검색과 임베딩에 맞는 입력으로 변환
  • 저품질 문서 필터링: 특수문자 비율, 영문 비율, 빈 문서 여부를 기준으로 제외
  • 길이 필터링 후 저장: 원문과 정제 문장의 길이를 모두 확인한 뒤 최종 CSV에 기록

이 과정에서 핵심은 외부 PDF 파일을 페이지 단위 텍스트가 아니라 문장 단위로 저장하는 것입니다. 그래서 검색 인덱스, 청크 병합, 벡터 DB 적재 같은 후속 단계로 바로 연결할 수 있습니다.
